Chapter 10: Testing and Debugging

  • Errors, Exceptions, and Warnings
    • Errors are caused when something goes wrong in a program. Three Types usually.
      • System Error- Problem with the system or external devices that the program is interacting with.
      • Programmer Error- Program contains incorrect syntax or faulty logic or a typo.
      • User Error- Data incorrectly added by user, which the program can’t handle.
    • Exceptions
      • An error that produces a return value that can then be used by the program to deal with the error
      • Stack Trace- A sequence of functions or method calls that lead to the point where the error occured.
    • Warnings
      • Occurs if there is an error in the code, but it isn’t enough to crash the program. 
      • The program will continue to run incorrectly
  • The Importance of Testing and Debugging
    • Text your code rigorously, ensure you are not letting your JavaScript fail silently.
  • Strict Mode
    • Considers coding practices that were previously “poor style” as actual errors
    • Can also improve code clarity and speed
    • Strict mode is recommended
    • First line of JavaScript file: “ ‘use strict’; “
    • Linting Tools- used to test the quality of JavaScript code. They call out sloppy programming practices and syntax errors.
    • Feature Detection- making an if statement to check if something is supported
  • Debugging in the Browser
    • Finding where bugs are in the code.
    • The Trusty Alert- alert(), because it stops a program at certain points, so you can check what’s being returned.
    • Using the Console- console.log() which will return in the console.
    • Debugging Tools- debugger; keyword creates a break point in the code that will pause the code.
    • Remove any code that was just for debugging before “shipping”
  • Error Objects
    • Const error = new Error(‘Oops something went wrong!’);
    • EvalError- not used currently
    • RangeError- when a number is outside an allowable range of values.
    • ReferenceError- reference is made to an item that doesn’t exist.
    • SyntaxError- Error in the syntax.
    • TypeError- Error in the type of value used (a string used when a number is expected)
    • URIEroor- Problem encoding or decoding the URI
    • InternalError- non-standard. When an error occurs in the JavaScript engine.
    • All of the above can also be used as a constructor to create custom error objects
  • Throwing Exceptions
    • Manually thrown excpetions, use throw statement.
    • Causes the execution of the code to stop
    • Best Practice- Throw an error object (throw new Error();)
  • Exception Handling
    • Exception occurs then program terminates with error message
    • Ideal in development, not great for production
    • Try, catch, and finally blocks
    • Do lots of testing, using functions
    • Test Driven Development (TDD)- Considered Best Practice
      • Write Tests the Intially fail
      • Write code to pass the tests
      • Refractor the Code
      • Test the refractories Code
      • Write more tests for new features 
  • Testing Frameworks
    • Jest- TDD framework

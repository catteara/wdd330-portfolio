Chapter 5: Objects

  • Object Literals 
    • Object- is a self-contained set of related values and functions.
    • Method- When a property’s value is a function
    • Object Literal- an object created directly in the languages by wrapping all its properties and methods in curly braces{}
      • Allow objects to be created quickly w/o the need for defining a class
      • Way to organize your code
  • Creating ObjectsEmpty Variable Example
    • Const spiderman = {};
    • Using a constructor function example (empty)
      • Const spiderman = new Object();
      • Not Recommended
    • Accessing Properties examples
      • Superman.name
        • More common
      • Superman[‘name’]
        • Only way to access nonstandard property and method names (Names that don’t follow the variable rules)
        • Let’s you evaluate an expression and use it as the property key
    • Checking if properties or methods exist examples
      • ‘City’ in Superman; >>false
      • Superman.city !== undefined; >>false
      • Superman.hasOwnProperty(‘city’); >>false
      • Superman.hasOwnProperty(‘name’); >>true
  • Nested Objects
    • Keyword this
      • Refers to the object that is within
      • Example
        • Const dice = { sides:6, roll() { return Math.floor(this.sides * Math.random()) + 1; }}
  • Namespacing
    • Object Literal Pattern 
      • Use to create a namespace for groups

Chapter 6: The Document Object Model

  • DOM (Document Object Model)
    • Represents an HTML document as a network of connected nodes that form a tree-like structure.
    • Treats everything on a web page as a node.
    • The HTML tag is the root tag and everything else is a child of this tag.
    • Not actually part of JavaScript, it is a language agnostic.
    • Can be used in any programming language.
    • Used in JavaScript to access and modify different parts using built-in object, document.]
  • Legacy DOM Shortcut Methods
    • Document.body - returns the body element of web page
    • Document.images - returns a node list of all the images contained in the document
    • Document.links - returns a node list of all ‘<a>’ elements and ‘<area>’ elements that have a href attribute
    • Document.anchors - returns node list of all ‘<a>’ elements that have a name attribute
    • Document.forms - returns node list of all forms in document 
  • Node lists are not arrays
  • Getting an Element
    • By ID - .getElementById(‘title’);
    • By Tag Name - .getElementsByTagName(‘li’);
    • By Class Name- .getElementsByClassName(‘hero’);
  • Query Selectors
    • .querySelector(‘’) method allows you to use CSS notation to find the <strong>first</strong> element in the document that matches a CSS selector. If no matches, null.
  • Setting an Elements Attribute Example 
    • WonderWoman.setAttribute(‘class’,’villain’); <<undefined
    • WonderWoman .getAttribute(‘class’); << “villain”

Chapter 7: Events

  • Common Types of Events
    • Mouse Events - mouseover(sounds like hover in CSS), mousedown, mouseup, click, dblclick (don’t use click and dblclick on the same element)
    • Keyboard Events - keydown, keypress, keyup (Keep modifier keys in mind)
    • Touch Events - touchstart, touchend, touchmove, touchenter, touchleave, touchcancel (No swipe, use combo of start, move, and leave). Touch Events are Complex.

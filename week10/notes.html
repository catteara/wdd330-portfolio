MDN: Client-side Form Validation

  • Client-side validation is an initial check and an important feature of good user experience
  • If it gets to the server and is then rejected, a noticeable delay is caused by a round trip to the server and then back to the client-side to tell the user to fix their data
  • Client-side validation is easy to get around, so server-side validation should also be used
  • Form Validation Examples
    • “This field is required”
    • “Enter a valid email address”
    • “Passwords should include at least 8 characters and at least one special character”
  • Types of client-side validations
    • Built-in form validation
      • HTML form validation. Better performance than JavaScript, but less customizable
      • Examples
        • required
        • minlength and maxlength- Constrains the length of entries
        • min and max
        • type- constrains the value of entries
        • pattern- Pattern uses regular expression
    • JavaScript
      • Completely customizable, but you have to make it or use a library
      • Constraint validation API (you can also do it yourself)
      • Implementing customized error messages

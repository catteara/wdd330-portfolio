Chapter 11: Further Functions

  • Function Properties and Methods
    • Functions are first-class objects.means they can have properties and methods.
    • Call and Apply Methods
      • call() method used to set the value of “this” inside a function to an object that is provided as the first argument
      • When the function called requires any parameters, provide these arguments after the first argument.
      • apply() method works the same except the arguments are provided as an array (even if there is only one argument)
    • Custom Properties 
      • You can add your own properties to functions like any object in JavaScript
      • Memoization
        • Saving the result in a cache property, to get returns quicker
  • Immediately Invoked Functions Expressions (IIFE __ ‘iffy’)
    • Anonymous function that is invoked as soons as it is defined. 
    • To do this: put parentheses at the end of the function and definition and around the whole declaration.
    • Use it to perform a task while still keeping variables under control.
    • Temporary Variables cause confusion, use with an IIFE.
    • Initializing Code- Use IIFE if code only needs to be run once.
    • Creates blocks of code
  • Self-Defining Functions
    • To do his assign an anonymous function to a variable that has the same name as the function
  • Recursive Functions
    • A function that invokes itself until a certain condition is met
  • Callbacks
    • Functions passed to other functions as arguments and then invoked within the function they are passed to.
  • Promises
    • Represents the future result of an asynchronous operation
    • Don’t do anything that can’t be achieved with callbacks, but can help simplify the process (and avoid convoluted code that can result if you use multiple callbacks.
    • The Promise Life Cycle 
      • Called, Pending, Settled (Resolved or Rejected)
    • Creating a promise
      • Use a constructor function, takes a function (called an executor) as an argument, it initializes the promise and starts the asynchronous operation. It accepts two functions as arguments: resolve() is passes, reject() if fails.
    • Once a promise is settled you can use the then() method to establish the outcome.
    • Chaining Multiple Promises - Each Promise will begin when the previous promise has been settled
  • Async Functions (async keyword)
    • Allows you to write asynchronous code like it was synchronous
    • Achieved by using the await operator before an asynchronous function.
  • Functions that Return Functions
    • To return a function assign it to a variable
  • Closures
    • One of the most powerful JavaScript features
    • A closure is formed when the inner function is returned by the outer function, maintaining access to any variables declared inside the enclosed function.
  • Introduction to Functional Programming
    • A programming paradigm.
    • Uses pure functions
      • Depends only on values provided as arguments 
      • No side-effects
      • Given the same arguments the function always returns the same result
    • Pure function must contain
      • At least one argument
      • A return value
  • Currying
    • Process that involves partial application of functions 
    • A function is curried when not all arguments have been supplied to the function, so it returns another function that retains the arguments provided
    • A final result is only returned once all expected arguments have been provided.
    • Relies on higher-order functions (ex: power() function)
    • Allows you to turn a single function into a series of functions instead

Chapter 11: Ajax

  • Clients and Servers
    • Javascript was designed as a client side scripting language, so it runs locally on the browser.
    • Ajax allows JavaScript to request resources from a server on behalf of the client.
    • A server is required when requesting resources using Ajax (this involves server-side language like PHP, Ruby, Node.js, or .NET)
  • Communicating with the server using the Fetch API
    • fetch() method, with the URL of the resource within quotes in the parentheses. 
    • Response Interface
      • Deals with the object that’s returned when the promise is fulfilled.
    • Redirect() method
      • Used to redirect to another URL.
    • Responses: text(), blob(), json(), and creating them using constructor function
    • Request Interface 
      • Request objects are created using Request() and the following properties: url, method, headers, mode, cache, credentials, redirect

Chapter 8: CSS3 Transforms and Transitions

  • Transforms
    • The transform property allows you to translate, rotate, scale, and/or skew any element on the page.
    • The value of the transform property is one or more transform function (separated by spaces) that are applied in the order they are listed.
      • Choose your ordering carefully
  • Translation 
    • Allows you to move elements left, right, up, or down (similar to the behavior of position: relative;)
    • When using a translation function, you’re moving elements without impacting the flow of the document.
    • A translated element can only be moved relative to its current position.
    • Transforms don’t work on inline elements (the solution they used was to change to display: inline-block;)
  • Scaling
    • Scale(x,y) function scales an element by the defined factors horizontally then vertically.
    • If only one value is provided it will be used for the x and y.
    • Has no impact on document flow
      • If this is an issue, consider the use of height, width, or font-size.
  • Rotation 
    • Rotates an element around the point of origin by a specified angle value
  • Skew
    • Skew(x,y) function specifies a skew along the x and y axes
    • If there is only one value, the skew will only take place on the x axis
  • Changing the Origin of the Transform
    • Transform-origin property. Uses the same syntax as background-position property
  • Transitions 
    • Allows the values of CSS properties to change over times (essentially providing simple animations) 
    • CSS transitions are declared along with the regular styles on an element
    •  Steps:
      • 1. Declare the original state of the elements in the default style declaration.
      • 2. Declare the final state of your transitioned element
      • 3.Include the transition functions in your default style declaration using the translation properties (transition-property, transition-duration, transition-timing-function, and transition-delay)
  • Transition-property 
    • Defines the CSS properties of the element that should be transitioned. 
    • If you want to transition more than one property comma separate them.
  • Transition-duration 
    • Sets how long the transition will take to go from default state to transitioned state
    • Seconds (s) or milliseconds(ms)
      • 2s or 200ms is standard
  • Transition-timing-function
    • Let’s you control the pace of the transition
    • Common timing functions: linear, ease-in, ease-out, ease-in-out
  • Transition shorthand 
    • Transition: -property -duration -timing-function -delay;
    • Example… 
      • transition; transform 0.2s ease-out 50ms;
  • Multiple Transitions 
    • Transition properties allow for multiple transitions in one call
  • Keyframes
    • To animate an element in CSS, first create a named animation, then attach it to an element in that element’s property declaration block.
    • Each key frame looks like its own nested CSS declaration block 
    • There are two keyterms — from and to — which evaluate to 0% and 100%.
  • Animation Properties 
    • Animation-name
    • Animation-duration
    • Animation-timing-function
    • Animation-iteration-count
    • Animation-direction
    • Animation-delay
    • Animation-fill-mode
    • Animation-play-state
  • Shorthand animation property
    • List information in order above, separated with spaces, if declaring multiple animations, separate with comma. 

Chapter 12: Canvas, SVG, and Drag and Drop

  • Canvas 
    • Set width and height in the canvas element
    • Starting to draw (in JS)
      • Var canvas = document.getElementById(“canvasId”); var context = canvas.getContext(“2d”);
    • Select Colors
      • .strokeStyle
      • .fillStyle
      • .fillRect
      • .strokeRect
    • Drawing a circle
      • Context.beginPath(); context.arc(50,50,30,0, Math.PI*2, true); context.closePath;
      • Arc (x, y, radius, startAngle, endAngle, anticlockwise(t/f))
    • Saving Canvas Drawings
    • Invisible to Screen Readers
    • Loads faster than SVG
    • Better than SVG if 
      • Painting pixels to the screen
      • No concerns about the ability to retrieve and modify your shapes
  • SVG (Scalable Vector Graphics)
    • Drawing a circle
      • <svg> <circle cx=“50” cy=“50” r=“25” fill=“color”/> </svg>
    • If you want a complex shape, you can use an image editor or free-use SVGs from online.
    • More accessible than canvas 
    • Better than Canvas if
      • Need the ability to access and change specific aspects of your graphics
  • Drag and Drop 
    • Two major kinds of functionality
      • Dragging files from your computer into a web page
      • Dragging elements into other elements on the same page
    • Adding Drag and Drop to Page
      • Set the draggable attribute on any HTML elements you’d like to be draggable. (Draggable=“true”)
      • Add an event listener for the dragstart event on any draggable HTML elements
      • Add an event listener for the drag lover and drop events on any elements that you want to have accept dropped items

Variables

  • A value stored in memory
  • (let, const, or var) = value;
  • Const does not allow you to reassign a variable to another value.
  • Using const to declare the majority of your variables can help to avoid bugs
  • Var is outdated, if you aren’t using const, use let

Scope

  • Const and let are both block scoped
  • Global Scope
    • Any variable declared outside of a block
    • It is accessible anywhere in the program
    • Keep global variables to a minimum to avoid overwriting
  • Local Scope
    • Blocks can be created to create Local Scope
    • Put blocks within {}

Naming Variables

  • Avoid starting variables with $ and _ to avoid issues with library
  • Names are case sensitive
  • camelCaseIsProbablyBetter but underscore_is_ also_an_option
    • Be Consistent!

Arrays, Logic, and Loops

Array

  • An unordered list of values.

Array Properties and Methods

  • length- used to find the length of an array.
  • pop()- removes the last item from an array.
  • shift()- removes the first item in an array.
  • push()- appends a new value to the end of the array.
  • unshift()- appends a new item to the beginning of the array.
  • concat()- merge an array with one or more arrays.
  • join()- turn array into a string that comprises all the items and seperates them with commas.
  • slice()- creates a subarray.
  • splice()- removes items from an array than inserts new items in their place.
  • reverse()- reverses the order of an array.
  • sort()- sorts by alphabetical order (default). This changes the order permanently.
  • indexOf('value')- finds the first occurence of a value in an array.
  • includes('value')- returns boolean value if the value occures or not.

DRY

  • Dont Repeat Yourself.
  • Every part of your program should only be written once.

Arrow Functions

  • example: const square = x => x*x;
  • easily identifiable by "=>" symbols
  • Arrow Function Advantages
    • less verbose
    • don't need to be put into parentheses
    • if it is only one line, it doesn't need to be inside a block
    • return keyword isn't required
    • don't bind their own value of this to the function
